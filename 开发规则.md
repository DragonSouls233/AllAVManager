# MDCX 项目开发规则（AI 必须读取）

> 版本：v2.0
> 适用范围：所有 AI 开发的代码修改、新增、重构任务
> 核心原则：**优先复用成熟代码，最小化自研成本**
> 六大板块：JAV(有码) / JAV(无码) / FC2 / 国产 / PORNHub / 欧美

## 一、代码引用优先级

### 1.1 三级参考代码库

`G:\MDCX\.references\` 目录下的代码资产按成熟度分为三级：

| 级别 | 目录 | 来源 | 成熟度 | 适用场景 |
|------|------|------|--------|----------|
| **P0** | `GitHub\*` | 行业已验证的开源项目 | 高（生产级） | **默认首选** |
| **P1** | `本地\*` | 自研原型项目（VaultX/ReelSorter/PornSimilarityPlatform）| 中（架构参考） | 仅参考架构设计和模块拆分方案 |
| **P2** | `特殊项目\*` | 自研已验证的功能模块 | 中（已验证） | 特定功能的完整方案（海角/下载器/查重） |

### 1.2 选择优先级规则

```
IF  功能在 GitHub 目录有成熟实现 → 直接复用/适配 (P0)
ELIF 功能在 特殊项目 目录有已验证实现 → 按 GitHub 规范重构后复用 (P2)
ELIF 功能在 本地目录 有参考实现 → 提取架构设计思路，按 GitHub 规范重写 (P1)
ELSE → 自研开发，对齐 GitHub 代码规范
```

### 1.3 禁止行为

- 禁止直接复制 `本地\*` 目录的低成熟度代码到生产代码
- 禁止在 `GitHub\*` 有成熟方案时自研新轮子
- 禁止混合多种实现风格 — 同一功能只能有一种标准化实现

```
1. 功能拆解：列出各方案的完整功能点清单

3. 冲突识别：找出接口签名、数据格式、错误处理的冲突点
```

## 四、代码规范要求

### 4.1 架构分层

```
app/
  api/routes/      # 路由层 - 只做 HTTP 映射和参数校验
  services/        # 服务层 - 核心业务逻辑
  db/              # 数据层 - ORM 模型和数据库操作
  tasks/           # 任务层 - 异步任务和扫描器
  crawlers/        # 爬虫层 - 站点适配器和刮削逻辑
  utils/           # 工具层 - 通用工具函数
  config/          # 配置层 - 配置模型和管理
```

### 4.2 代码风格

- **类型注解**：所有函数参数和返回值必须标注类型
- **异常处理**：使用 try/except 包裹可能失败的 IO/网络操作
- **日志**：使用 app.utils.logger.get_logger(__name__)，不直接用 print
- **配置**：通过 app.config.manager.get_config() 读取，不硬编码
- **异步**：IO 操作用 async/await，CPU 密集型用 run_in_executor

### 4.3 参考来源标注

每个文件头部必须标注参考来源：

```python
"""
功能描述

参考来源：
- P0: mdcx-master/mdcx/llm.py (MIT License)
- P2: 特殊项目/提取/core/extractor.py (自研已验证)
- P1: VaultX/src/shared/... (仅参考架构,无代码复用)

整合说明：
- 生命周期管理: 参考 P0 mdcx
- 提取流程: 参考 P2 提取目录
- 新增: 批量刮削、全局单例
"""
```

如果功能是从某来源直接复制的，需标注来源和许可协议。

## 六、内置代理规则（强制执行）

### 6.1 核心原则

**所有与刮削相关的网络请求必须通过内置代理（Xray）发起。** 代理是 MDCX 的基础设施，不是可选项。

### 6.2 适用范围

以下功能**必须使用内置代理**：

| 功能类别 | 具体场景 | 说明 |
|----------|----------|------|
| **所有模块的刮削器** | JAV/无码/FC2/国产/PORNHub/欧美的全部爬虫 | 任何 BaseCrawler 子类的网络请求 |
| **演员信息刮削** | IAFD/ThePornDB/JAVDB 演员页 | 演员元数据获取 |
| **封面/图片下载** | 封面图、演员头像、剧照 | 图片资源获取 |
| **搜索功能** | 各模块的搜索 API | 关键词搜索和结果抓取 |
| **CF 绕过** | Cloudflare 保护的站点 | curl_cffi + 代理双重绕过 |
| **海角社区提取** | 海角页面翻页、标题提取 | Selenium + 代理 |
| **下载器** | yt-dlp/M3U8/HTTP 下载引擎 | 下载流量也走代理 |
| **NFO/元数据补全** | 在线补全缺失字段 | API 请求走代理 |

### 6.3 代理架构

```
MDCX 内置代理 (app.services.proxy_manager)
  |
  ├── Xray 进程管理 (SOCKS5:127.0.0.1:18920 / HTTP:127.0.0.1:18921)
  ├── 多节点轮换 (proxy_manager loaded N nodes)
  ├── 健康检查 (自动检测可用节点)
  └── 故障切换 (连续3次失败自动切换节点)
  
使用方式：
  from app.services.proxy_manager import get_proxy
  proxy = get_proxy()  # 返回 {"http": "...", "https": "..."}
  
  # 在 httpx/aiohttp/requests 中使用
  async with httpx.AsyncClient(proxies=proxy) as client:
      ...
```

### 6.4 爬虫集成规范

所有新开发的爬虫**必须**在类属性中声明 `requires_proxy = True`，并在网络请求中注入代理：

```python
class MyCrawler(BaseCrawler):
    requires_proxy = True  # 强制声明
    
    async def scrape(self, code: str) -> Optional[ScrapeResult]:
        from app.services.proxy_manager import get_proxy
        proxy = get_proxy()
        async with AsyncHttpClient(proxy=proxy) as client:
            # 所有请求自动走代理
            html = await client.get_text(url)
```

### 6.5 禁止行为

- 禁止在刮削代码中直接使用 `httpx.AsyncClient()` 不带代理参数
- 禁止绕过代理管理器直接配置 hardcoded 代理地址
- 禁止在爬虫中设置 `requires_proxy = False`（除非有特殊原因并注释说明）
