## 一、概述

### 1.1 项目背景

MDCX 当前是以 **JAV 有码** 为主的视频管理系统，拥有完整的爬虫引擎（55+ 爬虫）、数据库模型、API 路由、前端界面和文件管理能力。为支持多种类型成人内容的统一管理，需要扩展为 5 模块架构：

| # | 模块 | 类型标识 | 说明 |
|---|------|----------|------|
| 1 | **JAV 有码** | `jav` | 已有，不动 |
| 2 | **JAV 无码** | `uncensored` | 新建（HEYZO/1pondo/Carib/无码数字番号） |
| 3 | **FC2** | `fc2` | 新建（FC2-PPV 编号体系） |
| 4 | **国产** | `chinese` | 新建（麻豆/天美/精东/91 等中文内容） |
| 5 | **PORNHub** | `pornhub` | 新建（欧美站点、Pornhub/XVideos 等） |

### 1.2 核心设计原则（v2.0 更新）

1. **零侵入** — 不修改现有 JAV 有码模块的数据库模型、API、前端代码
2. **独立数据库** — 每个模块使用独立 `.db` 文件，完全隔离（v2.0 从单表 type 改为独立库方案）
3. **独立目录挂载** — 每个模块独立配置媒体目录
4. **共享层统一** — 播放器、Cookie/代理、配置文件等共用
5. **渐进实现** — FC2 → JAV 无码 → 国产 → PORNHub 顺序
6. **国产文件夹即演员** — 国产模块独有：文件夹名作为演员名（v2.0 新增）
7. **渐进完善** — 先实现基础功能（扫描/导入/播放），再逐步增强刮削

## 二、数据库设计方案（独立数据库方案）

### 2.1 方案选择：独立数据库 vs 单表 type 字段

**v1.0 方案回顾**：原设计使用单表 `movies` + `type` 字段区分模块。

**v2.0 决策：采用独立数据库方案**（参考 OpenAver 模块化设计 + Stash GraphQL 分库）

**理由：**
1. **完全隔离** — 国产模块没有标准化番号，数据模型差异大，混用会导致 JAV 有码查询变慢
2. **风险控制** — 国产/FC2 的数据质量较低，不会影响已有的 JAV 有码库
3. **独立迁移** — 各模块可独立进行数据库迁移和版本管理
4. **灵活扩展** — 每个模块可独立选择存储引擎或优化策略
5. **用户清晰** — 用户能直观看到每个模块的数据库文件

## 四、国产模块特别设计：文件夹名 = 演员名

### 4.1 功能概述

**痛点**：国产内容（麻豆/天美/精东/91等）没有标准元数据源，刮削困难，演员信息几乎无法自动获取。

**解决方案**：利用用户的文件夹组织结构，直接将**文件夹名作为演员名**。用户只需按演员名建文件夹，系统自动将文件夹名映射为演员字段。

### 4.2 典型的文件夹结构

```
E:/Media/Chinese/
├── 梁佳芯/                      ← 文件夹名 = 演员名
│   ├── MD0269-梁佳芯-换妻性爱.mp4
│   └── MD0270-梁佳芯-七夕献礼.mp4
├── 唐芯/
│   ├── TM0165-唐芯-妈妈的爱.mp4
│   └── TM0166-唐芯-生日惊喜.mp4
├── 91制片厂/                     ← 文件夹可能不是演员名（工作室名）
│   ├── 田恬/
│   │   └── 91CM-081-田恬-继母.mp4
│   └── 张雅/
│       └── 91CM-082-张雅-护士.mp4
└── 未分类/                       ← 黑名单过滤，不识别为演员
    └── 一些散文件.mp4
```

### 4.3 多层文件夹识别策略

```
路径深度分析：
E:/Media/Chinese/梁佳芯/MD0269.mp4
                  ↑ actor_folder_depth=1 → actor=梁佳芯

E:/Media/Chinese/91制片厂/田恬/SD001.mp4
                           ↑ actor_folder_depth=1 → actor=田恬

E:/Media/Chinese/梁佳芯/2024/MD0269.mp4
                                       ↑ 需要 depth=2 才能追溯到梁佳芯
```

**策略优先级：**

```
1. 直接父文件夹（depth=1）：
   - Chinese/梁佳芯/MD0269.mp4 → actor = 梁佳芯
   - 成功率最高，推荐默认 depth=1

2. 上两级文件夹（depth=2）：
   - Chinese/工作室/演员/MD0269.mp4 → actor = 演员
   - 适用于有工作室套娃结构的用户

3. 多层级回退：
   - 从文件向上一级一级尝试，跳过黑名单后第一个有效名
   - Chinese/A/B/C/MD0269.mp4 → 从 C 开始试，跳过黑名单后找到 B = 演员
```

### 4.4 演员名智能清洗

从文件夹名提取演员名时，需进行智能清洗：

```python
import re

def extract_actor_from_folder(folder_name: str) -> list[str]:
    """
    从文件夹名提取演员名（国产模块专用）
    
    规则：
    1. 过滤黑名单文件夹名
    2. 排除已知工作室名（麻豆传媒、天美传媒等）
    3. 纯字母名保留（如 Xiangzi）
    4. 中文名提取（2-8个中文字符）
    5. "xxx+xxx" 格式分割为两个演员
    """
    BLACKLIST = {"新建文件夹", "合集", "精选", "unknown", "未分类",
                 "精选集", "我的收藏", "tmp", "temp", "downloads"}
    
    STUDIO_NAMES = {"麻豆传媒", "天美传媒", "果冻传媒", "精东影业",
                    "糖心VLOG", "蜜桃传媒", "星空无限", "SWAG"}
    
    if folder_name in BLACKLIST:
        return []
    if folder_name in STUDIO_NAMES:
        return []
    
    # 去除番号前缀（如 MD-0269.梁佳芯 → 梁佳芯）
    cleaned = re.sub(r'^[A-Z]+-?\d+[._\s]?', '', folder_name)
    
    # 按分隔符拆分多个演员名
    actors = re.split(r'[.+_&,，、]', cleaned)
    
    # 过滤不符合条件的
    result = []
    for a in actors:
        a = a.strip()
        if not a:
            continue
        # 纯中文名（2-8字）
        if re.match(r'^[\u4e00-\u9fff]{2,8}$', a):
            result.append(a)
        # 纯英文名（首字母大写）
        elif re.match(r'^[A-Z][a-z]+$', a):
            result.append(a)
        # 中英混合（如 梁佳芯Liang）
        elif re.match(r'^[\u4e00-\u9fff]{2,}[A-Za-z]+$', a):
            result.append(a)
    
    return result if result else [folder_name]  # 兜底返回原始名
```

## 五、爬虫层设计（v2.0 增强）

### 5.1 从 50+ 参考项目提取的爬虫优化策略

#### 5.1.1 多队列并发管道架构（来自 jav-scrapy）

```
[队列管理器]
     │
     ├── Queue 1: 索引页面扫描 (concurrency=3)
     │   → 获取搜索列表，提取详情页 URL
     │
     ├── Queue 2: 详情页抓取 (concurrency=5)
     │   → 解析元数据，提取封面 URL
     │
     ├── Queue 3: 图片下载 (concurrency=2)
     │   → 下载封面/样图（限速，防封）
     │
     └── Queue 4: 数据库写入 (concurrency=1)
         → 串行写入，防止并发冲突
```

**优点：**
- 各阶段独立并发控制
- 支持优雅关闭（SIGTERM → 排空队列 → 保存状态）
- 错误隔离（一个队列��败不影响其他）

### 8.1 视频感知哈希去重（优先级：高）

```python
import imagehash
from PIL import Image
import ffmpeg

def get_video_hash(video_path: str) -> str:
    """计算视频感知哈希（用于去重）"""
    # 提取关键帧
    out, _ = (
        ffmpeg.input(video_path, ss="00:00:10")
        .filter("select", "gt(scene,0.4)")
        .output("pipe:", format="image2pipe", vframes=5)
        .run(capture_stdout=True, capture_stderr=True)
    )
    # 计算帧哈希
    frames = [Image.open(BytesIO(frame)) for frame in out]
    hashes = [str(imagehash.phash(frame)) for frame in frames]
    return hashlib.sha256("".join(hashes).encode()).hexdigest()[:16]
```

**核心设计（来自 mdcx llm.py）：**

```python
class LLMClient:
    """LLM 刮削客户端"""
    
    def __init__(self):
        self.client = AsyncOpenAI(
            api_key=config.llm.api_key,
            base_url=config.llm.base_url,  # 兼容 Ollama/Gemini/DeepSeek
        )
        self.limiter = AsyncLimiter(
            max_rate=10,     # 每秒最多10次
            time_period=1
        )
        self._ref_count = 0  # 引用计数管理
    
    async def scrape(self, filename: str) -> dict:
        """使用 LLM 从文件名提取元数据"""
        async with self.limiter:
            response = await self.client.chat.completions.create(
                model=config.llm.model,  # 可配置
                messages=[
                    {"role": "system", "content": self.SYSTEM_PROMPT},
                    {"role": "user", "content": self._build_prompt(filename)},
                ],
            )
            return self._parse_response(response)

class DedupEngine:
    """三层视频去重引擎"""
    
    def find_duplicates(self, video_path: str) -> list[DuplicateInfo]:
        results = []
        
        # 第一层：文件哈希（最快，精确匹配）
        file_hash = self._get_file_hash(video_path)
        results += self._find_by_file_hash(file_hash)
        
        # 第二层：感知哈希（相似匹配，可查不同格式）
        phash = self._get_video_phash(video_path)  # 参考 fdupves
        results += self._find_by_phash(phash, threshold=0.8)
        
        # 第三层：音频指纹（参考 ReelSorter chromaprint）
        audio_hash = self._get_audio_fingerprint(video_path)
        results += self._find_by_audio(audio_hash)
        
        return self._deduplicate_results(results)
```

## 十二、风险与注意事项

### 12.1 国产模块风险

- **文件夹名冲突**：不同文件夹可能对应同一演员（如"梁佳芯"和"Liang Jiaxin"），需同名识别
- **文件夹改名**：用户改名后历史数据需刷新
- **混合文件夹**：一个文件夹内含多个演员的作品，无法精确分配
- **工作室文件夹**：用户可能以工作室名建文件夹（如"麻豆传媒"），需与演员名区分
- **无标准化番号**：降级策略需覆盖所有常见命名格式

### 12.2 PORNHub 模块风险

- **强反爬虫**：CloudFlare 5 秒盾 + 年龄验证弹窗 + 验证码
- **noindex/noarchive**：部分页面无法直接索引
- **API 限制**：无公开 API，完全依赖 HTML 解析
- **元数据不足**：部分视频缺少评分、演员信息

### 12.3 FC2 模块风险

- FC2 是 P2P 销售平台，没有官方元数据站
- 第三方站靠用户提交，覆盖不全
- FC2 番号很长（FC2-1234567），且部分视频只有纯数字 ID
- 演员信息经常缺失（素人内容为主）

### 12.4 向后兼容

- 已有 `scraper.db` 完全不动，仅读不写
- 旧前端无模块参数时默认显示所有模块数据
- 现有 Movie 模型不做任何修改
- 国产模块完全使用独立数据库，零影响

## 附录 B：国产模块文件夹演员完整设计

### B.1 核心数据流

```
用户文件夹结构:
E:/Media/Chinese/
├── 梁佳芯/                → extract → actors: [梁佳芯]
│   ├── MD0269-梁佳芯-换妻.mp4
│   └── MD0270-梁佳芯-献礼.mp4
├── 唐芯+王小妮/           → extract → actors: [唐芯, 王小妮]
│   └── TM0165-唐芯-妈妈.mp4
└── 未分类/                → BLACKLIST → 跳过
    └── random.mp4

数据库写入:
chinese.db
├── movies
│   ├── code=MD0269, title=..., folder_based_actors=["梁佳芯"]
│   └── code=TM0165, title=..., folder_based_actors=["唐芯","王小妮"]
├── actors
│   ├── name=梁佳芯, movie_count=2, source=folder
│   └── name=唐芯, movie_count=1, source=folder
```

### B.2 演员合并策略

当文件夹演员 + 文件名演员 + 刮削演员同时存在时：

```
优先级（从高到低）:
1. 刮削演员（如果有且匹配） → 可信度最高
2. 文件夹演员（由用户目录结构保证） → 几乎 100% 准确
3. 文件名演员（extract_actor_from_filename） → 中等准确
4. 三源合并（无冲突时全部保留，冲突时降权处理）
```

### B.3 配置示例

```yaml
# config.yaml 国产模块配置
modules:
  chinese:
    enabled: true
    media_dirs:
      - "E:/Media/Chinese"
    actor_from_folder:
      enabled: true                  # 开启文件夹演员
      folder_depth: 1                 # 父文件夹深度
      auto_create_actor: true         # 自动创建演员记录
      sync_on_scan: true              # 扫描时自动同步
      blacklist:                      # 过滤文件夹名
        - "新建文件夹"
        - "合集"
        - "精选"
        - "未分类"
        - "tmp"
        - "temp"
        - "downloads"
      studio_names_as_folder: false   # 工作室名是否作为文件夹名（如 麻豆传媒）
    scraper:
      enabled: true
      sources:
        - madouqu
        - love6
        - hscangku
```
